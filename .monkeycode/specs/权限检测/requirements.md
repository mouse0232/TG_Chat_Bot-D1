# 权限检测功能 - 需求文档

## 1. 功能概述

为 Telegram 双向机器人增加权限检测功能，允许管理员通过管理面板一键检测 Bot 自身所需权限和群管理所需权限是否已开通。如权限未开通，给出相应的提示信息，帮助管理员快速排查权限配置问题。

## 2. 功能范围

### 2.1 本次实现范围

- Bot 自身权限检测：验证 Bot Token 是否有效、Bot 基础 API 调用权限
- 群管理权限检测：验证 Bot 在管理群组中的管理员权限、话题管理权限
- 管理面板入口：在现有控制面板中增加"权限检测"入口
- 权限状态展示：以清晰的 UI 展示各项权限的开通状态
- 缺失权限提示：针对未开通的权限提供具体的解决建议

### 2.2 未来扩展

- 自动修复权限：检测后自动申请所需权限
- 权限变更通知：权限变动时主动通知管理员
- 权限历史记录：记录权限检测历史

## 3. 核心设计原则

### 3.1 零侵入性

权限检测功能独立于现有消息处理流程，不影响正常的消息转发、验证、反骚扰等核心功能。

### 3.2 即时反馈

检测操作应在 5 秒内完成并返回结果，避免管理员长时间等待。

### 3.3 最小权限原则

检测过程仅读取权限状态，不修改任何配置或权限设置。

### 3.4 容错性

即使部分检测失败，也应展示已成功检测的权限状态，并提供错误原因说明。

## 4. 详细需求

### 4.1 Bot 自身权限检测

| 需求项 | 说明 |
|--------|------|
| Bot Token 有效性 | 验证 BOT_TOKEN 环境变量是否配置且有效 |
| Bot 基础信息获取 | 调用 getMe API 获取 Bot 基本信息 |
| Bot 命令设置权限 | 验证 setMyCommands 权限是否可用 |
| 消息发送权限 | 验证 sendMessage 基础权限 |

### 4.2 群管理权限检测

| 需求项 | 说明 |
|--------|------|
| 管理群组配置检查 | 检查 ADMIN_GROUP_ID 是否配置 |
| Bot 管理员身份验证 | 验证 Bot 是否为管理群组的管理员 |
| 话题创建权限 | 验证 createForumTopic 权限 |
| 话题管理权限 | 验证话题内消息发送、置顶等权限 |
| 消息转发权限 | 验证 forwardMessage、copyMessage 权限 |

### 4.3 权限检测触发方式

| 触发方式 | 说明 |
|----------|------|
| 管理面板按钮 | 在控制面板增加"权限检测"按钮，点击触发检测 |
| 管理员命令 | 支持 /check_permissions 命令触发检测（可选） |

### 4.4 权限状态展示

| 状态类型 | 展示方式 | 说明 |
|----------|----------|------|
| ✅ 已开通 | 绿色勾号 + "已开通" | 权限检测通过 |
| ❌ 未开通 | 红色叉号 + "未开通" | 权限检测失败 |
| ⚠️ 部分受限 | 黄色警告 + 说明 | 权限存在但受限 |
| ⏳ 检测中 | 灰色时钟 + "检测中" | 正在检测该权限 |

### 4.5 缺失权限提示

| 缺失权限 | 提示信息 |
|----------|----------|
| Bot Token 无效 | "Bot Token 未配置或无效。请在 Cloudflare 环境变量中配置正确的 BOT_TOKEN" |
| 非管理员身份 | "Bot 不是管理群组的管理员。请将 Bot 添加为管理员并授予相应权限" |
| 话题创建权限缺失 | "缺少创建话题权限。请在群组管理中添加'创建话题'权限" |
| 消息发送权限缺失 | "缺少消息发送权限。请检查群组权限设置" |

### 4.6 配置项

无需新增配置项，复用现有环境变量：
- `BOT_TOKEN` - Bot Token
- `ADMIN_GROUP_ID` - 管理群组 ID

## 6. 非功能需求

### 6.1 性能要求

- 单次检测总耗时不超过 5 秒
- 并行检测可并行化的权限项
- API 调用超时设置为 2 秒

### 6.2 可靠性

- 检测失败不影响 Bot 正常运行
- 检测结果不缓存，每次点击实时检测
- 网络异常时显示明确的错误原因

### 6.3 可用性

- UI 简洁清晰，权限状态一目了然
- 错误提示具体可操作，避免模糊描述
- 支持重试机制，网络波动可重新检测

### 6.4 兼容性

- 兼容普通群组和超级群组
- 兼容话题模式和非话题模式
- Cloudflare Workers 环境兼容

## 7. 验收标准

- [ ] Bot Token 无效时明确提示配置错误
- [ ] Bot Token 有效时显示 Bot 用户名和 ID
- [ ] 未配置 ADMIN_GROUP_ID 时提示配置
- [ ] Bot 非管理员时提示添加管理员
- [ ] 缺少话题创建权限时明确提示
- [ ] 所有权限正常时显示完整通过报告
- [ ] 检测过程显示 Loading 状态
- [ ] 检测超时（>5 秒）显示超时错误
- [ ] 支持点击重新检测
- [ ] 管理面板入口位置合理（建议在"基础"或独立入口）
- [ ] 不影响现有消息处理流程
- [ ] 不产生额外的数据库写入操作
