# 反骚扰功能 - 需求文档

## 1. 功能概述

为 Telegram 双向机器人加入反骚扰功能，通过本地检测机制自动识别并拦截骚扰用户和消息，保护管理员免受垃圾信息、广告机器人等骚扰。

## 2. 功能范围

### 2.1 本次实现范围（本地检测）

- 用户身份检测（基于 Telegram Bot API 返回的用户信息）
- 消息内容检测（基于 Telegram Bot API 返回的消息结构）
- 可配置开关

### 2.2 未来扩展（云端检测）

- 云端 AI 检测服务
- 与本地检测并存

## 3. 详细需求

### 3.1 用户身份检测

| 检测项 | 条件 | 动作 |
|--------|------|------|
| 机器人账号 | `is_bot === true` | 提示"不符合聊天对象"，终止流程 |
| 空用户名 | `username` 为空或未设置 | 提示"不符合聊天对象"，终止流程 |
| Premium 用户 | `is_premium === true` | 放行，跳过其他检测 |

**优先级规则**：
- Premium 用户直接放行，跳过其他检测
- 机器人账号和空用户名用户提示不符合要求并终止流程
- **不进入黑名单，不调用 `manageBlacklist()`**

### 3.2 消息内容检测

| 检测项 | 条件 | 动作 |
|--------|------|------|
| Bot 转发 | `forward_from.is_bot === true` | 提示"不符合聊天对象"，**拉黑用户** |
| 内联键盘 | `message.reply_markup && message.reply_markup.inline_keyboard` 存在 | 提示"不符合聊天对象"，**拉黑用户** |
| @提及 | 消息中包含任意 `@` 提及（`msg.entities` 中存在 `mention` 或 `text_mention` 类型） | 提示"不符合聊天对象"，**拉黑用户** |

**检测时机**：
- 在已验证用户发送消息时进行检测
- 未验证用户不执行消息内容检测

**拉黑动作**：
- 设置 `is_blocked = true`
- 调用 `manageBlacklist()` 通知管理员
- 记录拉黑原因到 `user_info`

### 3.3 配置项

| 配置键 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `enable_anti_harassment` | boolean | `true` | 总开关 |
| `anti_harassment_block_bot` | boolean | `true` | 拦截机器人账号 |
| `anti_harassment_block_no_username` | boolean | `true` | 拦截空用户名用户 |
| `anti_harassment_allow_premium` | boolean | `true` | Premium 用户放行 |
| `anti_harassment_block_bot_forward` | boolean | `true` | 拦截 Bot 转发消息 |
| `anti_harassment_block_inline_keyboard` | boolean | `true` | 拦截带内联键盘的消息 |
| `anti_harassment_block_mention` | boolean | `true` | 拦截包含 @提及 的消息 |

## 4. 业务流程

### 4.1 用户首次交互流程

```
用户发送 /start 或消息
    │
    ▼
┌─────────────────┐
│ 反骚扰用户检测   │
│ (所有用户)      │
└────────┬────────┘
         │
    ┌────┴────────────┐
    ▼                 ▼
┌──────────┐   ┌──────────────┐
│ 通过      │   │ 触发规则      │
│          │   │ (bot/无用户名)│
└───┬──────┘   └──────┬───────┘
    │                 │
    ▼                 ▼
┌──────────┐   ┌──────────────────────┐
│ 正常流程  │   │ 提示"不符合聊天对象"  │
│ (验证等)  │   │ 终止流程              │
│          │   │ **不进入黑名单**       │
└──────────┘   └──────────────────────┘
```

### 4.2 已验证用户消息流程

```
已验证用户发送消息
    │
    ▼
┌─────────────────┐
│ 反骚扰消息检测  │
└────────┬────────┘
         │
    ┌────┴────────────────────┐
    ▼                         ▼
┌──────────┐           ┌──────────────┐
│ 通过      │           │ 触发规则      │
│          │           │ (bot转发/键盘)│
└───┬──────┘           └──────┬───────┘
    │                         │
    ▼                         ▼
┌──────────┐           ┌──────────────────────┐
│ 转发到话题│           │ 提示"不符合聊天对象"  │
│          │           │ **拉黑用户**          │
└──────────┘           │ 通知管理员           │
                       └──────────────────────┘
```

## 5. 非功能需求

### 5.1 性能要求

- 检测逻辑必须在 50ms 内完成
- 不应影响现有消息处理性能

### 5.2 兼容性

- 与现有验证流程兼容
- 与现有黑名单系统兼容
- 与现有管理面板兼容

### 5.3 可维护性

- 检测规则可独立扩展
- 配置项可通过管理面板调整

## 6. 验收标准

- [ ] 机器人账号 (`is_bot=true`) 收到"不符合聊天对象"提示，**不进入黑名单**
- [ ] 空用户名用户收到"不符合聊天对象"提示，**不进入黑名单**
- [ ] Premium 用户不被拦截
- [ ] Bot 转发的消息触发拦截（已验证用户），**进入黑名单**
- [ ] 带内联键盘的消息触发拦截（已验证用户），**进入黑名单**
- [ ] 包含 @提及 的消息触发拦截（已验证用户），**进入黑名单**
- [ ] 可通过配置关闭功能
- [ ] 与现有功能无冲突
